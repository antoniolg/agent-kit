# Platform Adaptation

Use this reference when `x-republish` needs to send the same X-origin post to multiple destinations.

## Goal

Keep the original post recognizable while avoiding obvious "copied from X" friction.

## Minimal adaptation

Use minimal adaptation when:
- the original text is already self-contained
- the target platform accepts short opinionated copy well
- the quote image carries most of the context

Typical changes:
- remove dangling handles or reply fragments
- expand one missing reference if the reader would otherwise be lost
- keep the structure almost intact

## Moderate adaptation

Use moderate adaptation when:
- the X post depends on timeline context
- the post uses shorthand that reads too abrupt elsewhere
- the target platform rewards slightly more framing

Typical changes:
- add one setup sentence
- make the cited idea explicit
- keep the close similar to the original unless it sounds unnatural

## Destination notes

### LinkedIn

Usually works best with:
- one clear opening claim
- a little more context than X
- the quote image as supporting proof, not as the whole post

Avoid:
- sounding like a raw repost
- opening with a handle or a platform-native aside

### X

When republishing to another X account or format:
- keep the text closest to the source version
- avoid over-expanding
- preserve punch and brevity

### Other PostFlow-managed platforms

Default rule:
- prefer the minimal viable adaptation first
- only expand when the post becomes unclear without the X timeline context

## Attribution

When the quoted post is central to the point:
- keep the author visible in the image when possible
- mention the source in the post text if the screenshot is too tight to show enough context

## Practical rule

If unsure, choose the smallest change that makes the post feel intentional on the destination platform.
